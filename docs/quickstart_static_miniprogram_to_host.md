# Quickstart: Static Mini-Program To Host App

This first-track guide creates a static two-screen mini-program named
`my_profile`, publishes it as public static artifacts, and opens it from a
Flutter host app.

No backend or middle-server is required in this guide. The host opens the
mini-program with:

- `appId`: `my_profile`
- `artifactBaseUrl`: the public URL where the static artifacts are hosted

Use Track 2 later when the mini-program needs dynamic data, auth, payments,
database access, files, secrets, or business logic.

## Stable Test Path

This guide is written for the current stable Windows flow:

- mini-program preview in Chrome
- host app run in Chrome
- host app run on Android emulator
- host app release APK build

Other Flutter devices may work, but this guide keeps the first path focused on
Chrome and Android emulator.

Install the CLI:

```powershell
dart pub global activate mini_program_tooling
miniprogram doctor
miniprogram --help
```

On Windows, make sure Dart pub global bin is on `PATH`:

```powershell
$env:Path += ";$env:LOCALAPPDATA\Pub\Cache\bin"
```

## 1. Create The Mini-Program

```powershell
cd D:\
miniprogram create my_profile --screen-format mp
cd D:\my_profile
```

The scaffold creates a portable Mp mini-program with:

- `manifest.json`
- `mp/program.dart`
- `mp/screens/my_profile_home.dart`
- `mp/screens/my_profile_details.dart`
- `tool/build_mp.dart`
- `pubspec.yaml`

Do not edit `mp/.build` directly. It is generated by `miniprogram build`.

## 2. Register Two Screens

Replace `mp/program.dart` with:

```dart
import 'package:mini_program_ui/mini_program_ui.dart';

import 'screens/my_profile_details.dart';
import 'screens/my_profile_home.dart';

final miniProgram = MpProgram(
  screens: <String, MpScreenBuilder>{
    'my_profile_home': buildMyProfileHome,
    'my_profile_details': buildMyProfileDetails,
  },
);
```

## 3. Edit The Home Screen

Replace `mp/screens/my_profile_home.dart` with:

```dart
import 'package:mini_program_ui/mini_program_ui.dart';

MpNode buildMyProfileHome() {
  return Mp.safeArea(
    child: Mp.padding(
      all: 16,
      child: Mp.column(
        children: <MpNode>[
          Mp.heading('Mehedi'),
          Mp.text('Mini-Program Platform Builder'),
          Mp.card(
            child: Mp.column(
              children: <MpNode>[
                Mp.text('Dhaka, Bangladesh'),
                Mp.text(
                  'Building lightweight mini-program experiences for Flutter host apps.',
                ),
              ],
            ),
          ),
          Mp.primaryButton(
            label: 'View Details',
            action: Mp.navigation.openScreen('my_profile_details'),
          ),
        ],
      ),
    ),
  );
}
```

This screen is static. It does not call an API.

## 4. Edit The Details Screen

Replace `mp/screens/my_profile_details.dart` with:

```dart
import 'package:mini_program_ui/mini_program_ui.dart';
```

to the previous mini-program screen.

## 5. Preview In Chrome

```powershell
miniprogram preview -d chrome
```

Check that:

- the home screen shows `Mehedi`
- `View Details` opens the details screen
- `Back` returns to the home screen

## 6. Build And Validate

```powershell
miniprogram build
miniprogram validate
```

Build writes deterministic screen JSON under:

```text
mp/.build/screens/
```

Validation checks the manifest and generated screen JSON.

## 7. Publish Static Artifacts

```powershell
miniprogram artifact build
miniprogram artifact verify
```

This writes portable static files under:

```text
D:\my_profile\artifacts
```

That folder is the portable artifact root. Copy or upload it to a public static
file host without changing its internal paths.

The artifact layout includes:

```text
artifacts/my_profile/latest.json
artifacts/my_profile/catalog.json
artifacts/my_profile/1.0.0/manifest.json
artifacts/my_profile/1.0.0/release.json
artifacts/my_profile/1.0.0/checksums.json
artifacts/my_profile/1.0.0/screens/<screenId>.json
artifacts/my_profile/1.0.0/assets/
```

## 8. Publish With GitHub Pages

GitHub Pages is the easiest first static host because most developers already
know GitHub.

1. Create a public GitHub repository named `my_profile_static`.
2. Upload or push `D:\my_profile\artifacts` into the repository root so the
   repository contains `artifacts/my_profile/...`.
3. In GitHub, open repository settings and enable Pages from the branch root. example: go to your root repo settings then click Pages button then Branch None change add main and then /root are okay then click save then need few minute for page can be live.
4. Use the GitHub Pages URL as the artifact base URL.

The URL usually looks like:

```text
https://<github-user>.github.io/my_profile_static/
```

The trailing slash is recommended. You can verify the publish by opening:

```text
https://<github-user>.github.io/my_profile_static/artifacts/my_profile/latest.json
```

Other static hosts work the same way. Any HTTPS host that serves static files
can serve the `artifacts` directory, including S3/object storage, Cloudflare Pages,
Netlify, Vercel, a CDN, or a normal static server. Track 1 does not need any
provider-specific setup beyond serving static files.

## 9. Create A Partner Package

Back in the mini-program folder:

```powershell
cd D:\my_profile
miniprogram partner package my_profile `
  --artifact-base-url https://<github-user>.github.io/my_profile_static/ `
  --output my_profile.partner.json
```

The partner package is a small JSON handoff file for host apps. It contains the
mini-program `appId` and `artifactBaseUrl`.

## 10. Create A Fresh Host App

Create a normal Flutter app that will open the mini-program:

```powershell
cd D:\
flutter create my_profile_host
cd D:\my_profile_host
```

The host app is separate from the mini-program. The host app installs the SDK
and opens mini-program artifacts by `appId`.

## 11. Initialize Mini-Program Host Integration

Run `embed init` inside the host app:

```powershell
miniprogram embed init --project-root .
miniprogram host endpoint import D:\my_profile\my_profile.partner.json --project-root .
```

`embed init` updates the host app automatically:

- adds `mini_program_sdk` and `mini_program_contracts` to `pubspec.yaml`
- creates generated adapter files under `lib/mini_program/`
- adds Android debug network files when the host has an Android target
- keeps the host app in control of where buttons/routes are placed

The generated `lib/mini_program/` folder normally contains:

```text
lib/mini_program/
  app_host_bridge.dart
  mini_program.dart
  mini_program_launcher.dart
  mini_program_runtime_setup.dart
  mini_program_endpoints.dart
  mini_program_registry.dart
```

Important generated files:

- `mini_program.dart` is the barrel import. It re-exports the SDK and generated helpers.
- `mini_program_runtime_setup.dart` creates `buildMiniProgramConfig(...)`.
- `mini_program_launcher.dart` creates `openAppMiniProgram(...)`.
- `mini_program_endpoints.dart` stores imported `artifactBaseUrl` endpoints.
- `mini_program_registry.dart` stores app IDs and titles for imported mini-programs.
- `app_host_bridge.dart` is where host-owned native actions can be wired later.

`host endpoint import` reads `D:\my_profile\my_profile.partner.json` and adds an
endpoint for `my_profile`. Opening still uses only `appId + artifactBaseUrl`.

## 12. Replace Host `main.dart`

For a fresh host app, replace `D:\my_profile_host\lib\main.dart` with this full
example:

```dart
import 'package:flutter/material.dart';

import 'mini_program/mini_program.dart';
import 'mini_program/mini_program_endpoints.dart';

void main() {
  runApp(
    MiniProgramScope(
      config: buildMiniProgramConfig(endpoints: buildMiniProgramEndpoints()),
      child: const MyProfileHostApp(),
    ),
  );
}

class MyProfileHostApp extends StatelessWidget {
  const MyProfileHostApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      title: 'My Profile Host',
      theme: ThemeData(
        colorScheme: ColorScheme.fromSeed(seedColor: Colors.teal),
        useMaterial3: true,
      ),
      home: const MyProfileHostHome(),
    );
  }
}

class MyProfileHostHome extends StatelessWidget {
  const MyProfileHostHome({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Mini-Program Host')),
      body: Center(
        child: Padding(
          padding: const EdgeInsets.all(24),
          child: Column(
            mainAxisSize: MainAxisSize.min,
            children: <Widget>[
              const Icon(Icons.account_circle_outlined, size: 72),
              const SizedBox(height: 16),
              const Text(
                'This Flutter app opens the public static my_profile mini-program.',
                textAlign: TextAlign.center,
              ),
              const SizedBox(height: 24),
              FilledButton.icon(
                onPressed: () {
                  openAppMiniProgram(
                    context,
                    appId: 'my_profile',
                    title: 'My Profile',
                  );
                },
                icon: const Icon(Icons.open_in_new),
                label: const Text('Open My Profile'),
              ),
            ],
          ),
        ),
      ),
    );
  }
}
```

Imports used by the host:

```dart
import 'mini_program/mini_program.dart';
import 'mini_program/mini_program_endpoints.dart';
```

Why these imports are needed:

- `mini_program.dart` exposes `MiniProgramScope`, `buildMiniProgramConfig`, and
  `openAppMiniProgram`.
- `mini_program_endpoints.dart` exposes `buildMiniProgramEndpoints()`, which maps
  `my_profile` to the GitHub Pages `artifactBaseUrl`.

How the integration works:

1. `MiniProgramScope` is placed above the app so any page can open mini-programs.
2. `buildMiniProgramConfig(endpoints: buildMiniProgramEndpoints())` gives the SDK
   the imported static artifact endpoints.
3. The button calls `openAppMiniProgram(context, appId: 'my_profile')`.
4. The SDK looks up `my_profile`, downloads `latest.json`, then downloads the
   screen JSON from the static artifact host.
5. The SDK renders the Mp JSON screen inside the host app.

No backend or middle-server is used in this flow.

## 13. Run The Host In Chrome

```powershell
cd D:\my_profile_host
miniprogram host run -d chrome --project-root .
```

You can also use Flutter directly:

```powershell
flutter run -d chrome
```

## 14. Add A Host Button In An Existing App

In a host app button, menu item, or navigation action, call:

```dart
openAppMiniProgram(
  context,
  appId: 'my_profile',
  title: 'My Profile',
);
```

The host app opens the mini-program by downloading the manifest and screen
artifacts from `artifactBaseUrl`.

For an existing app, the same rule applies: wrap the app with
`MiniProgramScope`, import the endpoint map, then call `openAppMiniProgram` from
your own button, drawer item, tab, or route.

## 15. Run Android And Build APK

Run on Android emulator:

```powershell
cd D:\my_profile_host
flutter run -d emulator-5554
```

Build a release APK:

```powershell
flutter build apk
```

Android build notes:

- Keep at least 5-8 GB free on `C:` for Gradle and Flutter build caches.
- If release icon tree-shaking reports missing Cupertino icons, add
  `cupertino_icons: ^1.0.8` to the host app dependencies.
- If a debug Android manifest reports a `usesCleartextTraffic` merge conflict,
  use the latest `mini_program_tooling` and rerun `miniprogram embed init
  --force`.

## VS Code Extension Flow

The VS Code extension runs the same CLI commands.

1. `MiniProgram: Create MiniProgram`
2. Edit `mp/program.dart`.
3. Edit `mp/screens/my_profile_home.dart`.
4. Edit `mp/screens/my_profile_details.dart`.
5. `MiniProgram: Preview`
6. `MiniProgram: Build`
7. `MiniProgram: Validate`
8. Run `miniprogram artifact build` and `miniprogram artifact verify`.
9. Publish the generated `artifacts` directory to GitHub Pages.
10. `MiniProgram: Create Partner Package`
11. Create or open the Flutter host app workspace.
12. `MiniProgram: Embed Init`
13. `MiniProgram: Import Host Endpoint`
14. Replace or update `lib/main.dart` with `MiniProgramScope` and a button.
15. `MiniProgram: Run Host App`

Configure `miniProgram.cliPath` only when the `miniprogram` command is not on
`PATH`.
